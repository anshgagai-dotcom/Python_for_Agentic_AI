## Repository Rules

1. **One folder per class** — Use `Week_XX/Class_XX/` naming consistently.
2. **Daily commits** — Push meaningful progress at least once per day.
3. **Clear commit messages** — e.g. `Week_00: Complete Class_01 homework and practice`.
4. **No secrets** — Never commit `.env` files, API keys, or credentials.
5. **Keep notes in Markdown** — Use `.md` for all written documentation.
6. **Code in `.py` and `.ipynb`** — Homework and practice in both formats when useful.
7. **Review professionally** — YouTube reviews should be constructive and specific.
8. **Extend cleanly** — Add new weeks following the same pattern as the internship progresses.

## Getting Started

```bash
# Clone the repository (SSH)
git clone git@github.com:anshgagai-dotcom/Python_Class.git
cd Python_Class

# Create a virtual environment (recommended)
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS / Linux

# Install dependencies (as they are added)
pip install -r requirements.txt

# Launch Jupyter (optional)
jupyter notebook
```
